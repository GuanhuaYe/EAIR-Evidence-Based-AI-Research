# Datasheet template (Gebru et al., adapted)

Auto-fill from paper + run logs. Flag any unanswered question — those
are submission blockers if venue requires the datasheet.

## Motivation
- Q1. For what purpose was the dataset created? Was there a specific
  task in mind? Was there a specific gap that needed to be filled?
- Q2. Who created the dataset and on behalf of which entity?
- Q3. Who funded the creation of the dataset?

## Composition
- Q4. What do the instances that comprise the dataset represent?
- Q5. How many instances are there in total?
- Q6. Does the dataset contain all possible instances or is it a
  sample? If a sample, what is the population, and is the sample
  representative?
- Q7. What data does each instance consist of?
- Q8. Is there a label or target associated with each instance?
- Q9. Is any information missing from individual instances?
- Q10. Are relationships between individual instances made explicit?
- Q11. Are there recommended data splits?
- Q12. Are there any errors, sources of noise, or redundancies?
- Q13. Is the dataset self-contained, or does it link to external
  resources?
- Q14. Does the dataset contain data that might be considered
  confidential?
- Q15. Does the dataset contain data that, if viewed directly, might
  be offensive, insulting, threatening, or might otherwise cause
  anxiety?
- Q16. Does the dataset relate to people?
- Q17. Does the dataset identify any subpopulations (e.g., by age,
  gender)?
- Q18. Is it possible to identify individuals from the dataset?
- Q19. Does the dataset contain data that might be considered
  sensitive in any way?

## Collection process
- Q20. How was the data associated with each instance acquired?
- Q21. What mechanisms or procedures were used to collect the data?
- Q22. If the dataset is a sample, what was the sampling strategy?
- Q23. Who was involved in the data collection process, and how were
  they compensated?
- Q24. Over what timeframe was the data collected?
- Q25. Were any ethical review processes conducted?

## Preprocessing / cleaning / labeling
- Q26. Was any preprocessing/cleaning/labeling done?
- Q27. Was the raw data saved in addition to the preprocessed data?
- Q28. Is the software that was used to preprocess/clean/label the
  data available?

## Uses
- Q29. Has the dataset been used for any tasks already?
- Q30. Is there a repository linking to any or all papers that use
  the dataset?
- Q31. What other tasks could the dataset be used for?
- Q32. Is there anything about the composition or collection process
  that might impact future uses?
- Q33. Are there tasks for which the dataset should NOT be used?

## Distribution
- Q34. Will the dataset be distributed to third parties?
- Q35. How will the dataset be distributed?
- Q36. When will the dataset be distributed?
- Q37. Will the dataset be distributed under a copyright or other IP
  license / TOU?
- Q38. Have any third parties imposed IP-based or other restrictions?
- Q39. Do any export controls or other regulatory restrictions apply?

## Maintenance
- Q40. Who will support / host / maintain the dataset?
- Q41. How can the curator be contacted?
- Q42. Is there an erratum?
- Q43. Will the dataset be updated? If so, by whom and on what schedule?
- Q44. If the dataset relates to people, are there applicable limits
  on retention of the data?
- Q45. Will older versions of the dataset continue to be supported?
- Q46. If others want to extend / augment / build on / contribute to
  the dataset, is there a mechanism for them to do so?
