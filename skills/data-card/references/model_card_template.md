# Model card template (Mitchell et al., adapted)

For released models. Skip if only releasing code or only releasing
data.

## Model details
- Name and version
- Type (architecture family, parameter count, modality)
- Date of release
- License
- Paper / repo / model-hub link
- Training compute (GPU hours, peak VRAM, total carbon if known)
- Authors / contact

## Intended use
- Primary intended uses
- Primary intended users (researchers, practitioners, downstream
  product teams)
- Out-of-scope uses

## Factors
- Relevant groups affected by model performance (demographic,
  linguistic, environmental)
- Instrumentation: what factors are measured during evaluation

## Metrics
- Metrics used (with justification)
- Decision thresholds (if classifier)
- Variation / disaggregation approach

## Evaluation data
- Datasets used
- Motivation for choosing them
- Preprocessing applied

## Training data
- Datasets used (with citations)
- Distribution / known biases
- Filtering / curation applied

## Quantitative analyses
- Disaggregated performance across groups / factors
- Confidence intervals
- Known failure modes

## Ethical considerations
- Data sensitivity (PHI, faces, scraped social media, minors)
- Potential misuse
- Mitigations applied
- Residual risks

## Caveats and recommendations
- Known unknowns
- Recommendations for downstream users
- Open questions for future research
